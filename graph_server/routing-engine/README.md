# Rescue-Mind Core Engine 🚀

This repository houses the backend infrastructure for the **Rescue-Mind** ecosystem, combining a high-performance Go API gateway with an intelligent, Python-based routing engine capable of integrating real-time Machine Learning prediction models.

---

## 🏗️ Architecture & Functions

### 1. Go API Gateway (`rescuemind-api`)
The central nervous system of the backend, written in Go for extreme concurrency and speed.
- **REST Controller (`/route`)**: Handles standard HTTP requests from external clients (like React Native mobile apps).
- **gRPC Client**: Communicates internally with the Python routing engine using lightweight, compiled Protocol Buffers (`routing.proto`).
- **WebSockets**: Real-time hub (`websocket_handler.go`) designed to stream live notifications or telemetry data down to connected clients.
- **Data Parsing**: Automatically maps the complex Protobuf routing paths into structured, frontend-ready JSON.

### 2. Python Routing Engine & ML Engine (`routing-engine` / Graph Model)
The heavy mathematical computing wing of the backend, built in Python to easily integrate with Machine Learning frameworks.
- **gRPC Server (`grpc_server.py`)**: A multi-threaded worker listening for routing tasks.
- **NetworkX Graph Memory**: Loads geographical geometries (`city_nodes.geojson`, `city_roads.geojson`) entirely into memory at startup.
- **KD-Tree (`kd_tree.py`)**: Solves the Nearest-Neighbor problem. Converts raw GPS coordinate requests (`lat,lon`) from the frontend into precise physical road intersections instantly.
- **Dynamic A* Pathfinding (`astar.py`)**: Computes optimal paths based on completely dynamic weights.

## 📡 Frontend Interaction & Responses

The frontend (e.g., React Native maps) will communicate strictly with the Go API Gateway over standard HTTPS.

**Endpoint**: `GET /route?start=LAT,LON&end=LAT,LON`

**Example Request**:
```http
GET http://localhost:8080/route?start=18.5274,73.8732&end=18.5200,73.8600
```

**JSON Response Payload**:
The server concurrently calculates three distinct optimal routes to provide maximum utility to the user.

## 🧠 ML Model Interaction (Dynamic Weights)

The routing engine is fully integrated with 2D Machine Learning prediction matrices (e.g., the 30x30 Gaussian blur disaster hazard outputs from the CNN classifier).

The system continuously scans for `data/blurred_matrix.csv`. If present, it maps the geographical bounding box of the city directly to the Matrix, allowing disaster "Hotspots" generated by the ML model to alter the routing algorithm instantly.

**How it works (`ml_provider.py` & `graph_updater.py`)**:
1. The CNN Model outputs a spatial risk float matrix exported as `blurred_matrix.csv`.
2. The `MLProvider` service scans the base geographic nodes (`city_nodes.geojson`) to establish extreme lat/lon bounding boxes.
3. It perfectly overlays the 30x30 CSV matrix onto the city map, calculating the `lat_step` and `lon_step` width of each grid cell.
4. The Routing Engine calculates exactly which matrix cell every physical road belongs to, extracting its hazard multiplier.
5. The engine inflates the road's baseline physical distance by `(1.0 + hazard_multiplier)`.
6. As the algorithm executes the `SAFEST` path route, it mathematically avoids roads that the ML model deemed dangerous.

```python
# The integration is automatically instantiated inside RoutingService on startup:
from services.ml_provider import MLProvider
from graph.graph_updater import update_graph_safety_weights

ml_provider = MLProvider()
if ml_provider.load_from_csv("data/blurred_matrix.csv"):
    ml_provider.compute_bounds(self.G)
    update_graph_safety_weights(self.G, ml_provider.get_matrix(), ...)
```

## 🏃 How to Run the Ecosystem

Because this architecture separates the web-server from the math-solver, you must run both components simultaneously.

### Step 1: Start the Python Backend
```bash
cd routing-engine
source venv/bin/activate

# Start the gRPC background worker (listening on port 50051)
python grpc_server.py
```

### Step 2: Start the Go API Server
Open a second terminal window:
```bash
cd rescuemind-api

# Start the public-facing HTTP server (listening on port 8080)
go run ./cmd/server
```

### Step 3: View Activity
Navigate your browser or `curl` to the Go API running on port `8080` to see the backend automatically relay requests to Python locally!
